修复XP和Vista双启动菜单

症状：装xp，vista，ubuntu三系统，后来ubuntu下vpn不太好用，而我又非需要vpn共享上网不可，于是在xp下直接格了ubuntu，导致开机Grub出现错误 用xp的故障回复台修复 于是开机就直接进入xp，没有之前的vista启动选择了

做法：

1.插入Vista光盘,或使用 Daemon Tools 加载 Vista 光盘像。

2.打开“运行” (开始〉运行), 输入 cmd。

3.输入 X: (X代表你的光驱盘符或虚拟光驱盘符)。

4.输入 cd boot。

5.输入 bootsect /nt60 SYS (删除是nt52) 。

6.重新启动，Vista 的启动管理器已经恢复。

【转载】09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 ——让我见识了 Web 2.0 用户的力量

原文链接：http://www.cnbeta.com/articles/25951.htm

你知道什么是 Web 2.0 吗？你可以看看我的文章《什么是 Web 2.0 ？》，今天 09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 这组号码终于让我真正地见识了 Web 2.0 的力量，用户或网民的力量。

Web 2.0 的代表网站 Digg 今天发生了一件大事， Digg 管理层在用户的力量下终于屈服，这绝对可以成为 Web 2.0 时代的象征性历史事件。

什么是 Web 2.0 ？

简单来说， Web 2.0 的网络就是由用户，也就是网民们生成（ generate ）的数据所组成的网络，其中的代表应用就有部落格（blog ），部落格的一切正是由网民自己生成的，Web 2.0 是身为用户的我们所主控的网络。

从去年到现在， Web 2.0 的网站不断冒出，时代杂志（ TIME Magazine ）更把全世界的网民们推选为 2006 年的风云人物，我们知道 Web 2.0 的时代来到了。

Web 2.0 的代表 Digg

Digg 是 Web 2.0 的代表性网站， Digg 上的资讯都是由用户所提交，再由所有的 Digg 用户投票决定什么资讯放在首页。

比如说，现在我发现了一则新闻，我想把它推荐给全世界，这时我就把这则新闻的网址提交到 Digg ，如果 Digg 的会员们喜欢的话，他们会投票给我提交的网址，当票数达到一定的数量，我所提交的这个网址，也就是这则新闻，就会被排列到首页，然后来自全世界的千千万万网民就可以与我一起分享这则新闻，这就是 Web 2.0 。

09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0

在 Digg 网站发生的大件事是由 09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 ，这组号码所引发。这组是什么号码呢？

大约在今年 2007 年 2 月的时候，骇客 amezami 在著名视频技术论坛 Doom9 对外宣布，他已经成功破解了新一代 DVD 格式 Blu-ray 和 HD DVD 的加密技术。 Blu-ray 和 HD DVD 的加密技术是电影企业们用来对付盗版的重要措施，这些世界企业寄予了厚望给这些加密技术，希望以此拯救全世界的影视工业。

破解加密技术的关键是一组号码 09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 ，这组号码就好像是密码，使用这组密码可以轻松解密 Blu-ray 和 HD DVD ，再直接地说就是，我们可以轻易制造出盗版了。

那么 09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 和 Web 2.0 和 Digg 有什么关系呢？

Digg 和 09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0

在今天下午，也就是马来西亚和中国时间的 2007 年 5 月 2 日下午，某某位 Digg 会员把有关上面这个 DVD 破解的消息提交到了 Digg ，短时间内，这个消息获得非常高的票数，自动排到了 Digg 首页。

这时候正是大事件的开始， Digg 的管理层因为赞助商或广告商的关系，以违反条规为由，把这个 DVD 被破解消息删除了，同时发布了部落格文章对这样的行为表态，《 What’s Happening with HD-DVD Stories? 》。

Digg 用户在发现了管理层的遮蔽行为后极为愤怒，认为这样阻止了言论自由，违反 Web 2.0 以用户为中心的原则，结果大家群情汹涌，群起蜂拥发起示威行动，部分 Digg 用户呼吁大家一起投票，把所有有关 HD DVD 的消息排到首页，让 Digg 首页完全充满 09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 这组号码。

Digg 完全被攻陷

来到这个时候，整件事情到来了沸点，来到了最高潮，在极短的时间内，用户们的力量完全显现了在 Digg 首页。

这是我在当时及时存储下来的截图（ screenshot ），大家可以从图中看到当时的 Digg 首页，每一项的提交都离不开 09 F9 11 02 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 这组号码，而且每一项的投票量都异常高，想当时我兴奋地亲眼目睹了 Web 2.0 用户的力量。

Digg 的用户们持续不断地进行示威行动，从事件开始后有关的提交不断涌现， Digg 首页不断更新，每一项更新都是 9D 74 E3 5B D8 41 56 C5 63 56 88 C0 ，没有间断。这个时候我用这句话来形容，“ Digg 已经被用户完全攻陷了”。

Digg 管理层的低头

经过一阵轰轰烈烈的示威，一段不长的时间， Digg 管理层终于认输，他们听从用户们的意愿，不再做任何损害用户利益的遮蔽行为，他们再次发布部落格文章表态，《 Digg This: 09-f9-11-02-9d-74-e3-5b-d8-41-56-c5-63-56-88-c0 》。

Digg 的创办人 Kevin Rose 在文章里说，他们听到了用户们的声音，“看到了上百篇文章和上千的留言，我们看清楚了你们的意愿，你们宁愿看到 Digg 去跟大企业对抗，我们听到了，从即刻起，我们不会再删除任何有关这组号码的文章提交和留言，我们会面对所有可能发生的后果。”

Digg 始终低头，在文章的最后，创办人 Kevin Rose 说，“ If we lose, then what the hell, at least we died trying ”。

Web 2.0 时代 用户就是网络

我非常庆幸，因为我目睹了用户的力量，目睹用户争取了自己的言论自由，目睹了用户的胜利，目睹了 Web 2.0 时代， Web 2.0 是用户的网络，是我们的互联网，我们网民就是互联网的核心。

最后，我非荣幸地，为这次极具意义的象征性事件写下这篇记录，让我们一起来见证这个 Web 2.0 的新世代。你看，我现在就正在运用用户的力量。

【转载】ubuntu下配置vpn例子一则

原文链接：搞定了ubuntu下学校图书馆的VPN

学校图书馆给住在校外的老师和研究生开设了VPN服务，这可是件大好事，免去了我们的奔波之苦，可惜图书馆的VPN指南中只介绍了Windows下的设置方法，今天我参考rhfcaesar的方法试着在ubuntu下搞了搞，居然成功了，哈哈，这下更坚定了我用Linux的信心。

具体方法如下：

1. 把下面一行加到/etc/apt/source.list

deb http://quozl.netrek.org/pptp/pptpconfig ./

别忘记了后面的./，然后

sudo apt-get install pptpconfig

会根据依赖关系安装大概10个包左右

2. 在终端输入sudo pptpconfig, 出现图形配置界面，作如下配置：

点击Server标签，填入以下各项

Name: 给你的VPN起个名字，随便填

Server: 219.239.104.3 这是师大的VPN IP地址

Domain: 空置不填

User Name: 填入借书证号

Password: 填入借书证密码

点击Routing标签, 选择All to Tunnel

点击DNS标签，去掉Automatic, 并空置下面两栏不填

点击 Encryption标签，去掉所有项

点击ADD，把刚刚做的设置加入VPN配置列表

3. 修改配置文件：

在终端执行sudo gedit /etc/ppp/options.pptp

用#号注释掉下面两句

refuse-chap

refuse-mschap

并将Encrption以下的行全部注释掉

4. 在终端中输入sudo pptpconfig，点击Start，哈哈，VPN连上了！试着访问一下图书馆的各种国外文献数据库，成功！

回头要把这个方法发给图书馆的相关负责人，让他们加到VPN User’s Guide中。

图书馆中的国外文献数据库都支持Firefox浏览器，试用了一下，一点问题都没有。查中文文献我只用CNKI，但CNKI升级后就不支持Firefox 了，其实就是一个验证登录的脚本不兼容—不用Javascript，非要用什么VBScript，垃圾！一定要向CNKI提意见。

至此，我的大部分工作都可以用ubuntu来做了，万不得已的情况下才用windows，不用盗版的感觉就是好。

【转载】Windows2003单网卡配置VPN服务器

原文地址：http://www.173wb.com//2007-01/4.html

1.       前期准备

首先，VPN服务器与系统自带的ICS服务冲突，在开始配置VPN服务器前，应当先关闭ICS服务，关闭ICS服务后，记得重启一下机器。

另外，很多杀毒软件的所谓火墙也会造成影响，如果发生问题，请考虑先暂时关闭杀毒软件测试或先仔细观察系统日志和杀毒软件日志。

2.       启用windows2003VPN服务器

管理工具->路由和远程访问

点击所属服务器，鼠标右键点击“配置并启动路由和远程访问”，而在后面的动作中，如果操作失败，可以随时点击“禁用路由和远程访问”以恢复到原始配置，重新再配置。

选择“自定义配置”

如果仅仅想VPN连接到服务器，选择“VPN访问”即可，如果想通过服务器NAT上网，还可以点击“NAT和基本防火墙”

服务配置完成，点击“是”开始服务

3.       配置windows2003VPN服务器

选择所属服务器，右键点击“属性”

选到“IP”页，再这里添加准备分配给VPN客户的IP，一般来说，除非你有三层权限，否则你当然只能分配私网IP给客户。在这里插一句，你要考虑 到有很多位于内网的客户使用的是例如“192.168.0.x”的私网IP，如果你也分配同段的私网IP，将会给客户带来一些麻烦，所以建议你使用一些不 常用的IP段。

选择所属服务器的“IP路由选择”->“静态路由”，右键点击“信件静态路由”，本页默认应该是空的

“接口”选择物理网卡名称，“网关”填写本机“默认网关”(本示范用的是内网，你在公网实际操作的事后应该填写你服务器的公网网关IP地址)，“目标”和“网络掩码”皆填写“0.0.0.0”

选择所属服务器的“DHCP中继代理程序”，删除默认的“内部”接口

为“DHCP中继代理程序”新增一个针对本地物理网卡的接口

选择本地物理网卡

确定

为“NAT/基本防火墙”新增NAT接口

选择本地物理网卡

在“NAT/基本火墙”页，选择“公用接口连接到Internet”和勾选“在此接口上启用NAT”

至此VPN服务器配置完成

4.       配置用户示范

新加一个用户

在“拨入”属性页的“远程访问权限(拨入或VPN)”属性，勾选“允许访问”即可允许该用户登陆VPN服务器

完成图，用户拨接上服务器后，即可在“路由和远程访问”管理器上以及“联网”属性页看到该虚拟连接

后记：本文没有讨论到如何利用RADIUS服务管理拨入管理以及DHCP管理拨入用户的IP。

[转载]在ubuntu中安装微软雅黑字体

http://forum.ubuntu.org.cn/viewtopic.php?t=21372&postdays=0&postorder=asc&start=0

看了好多对中文字体的美化的贴，但我对我机器上的字体是这样美化的，希望对新手有点用处。请老手能见笑了。

下载华文黑体字体或微软雅黑字体(0.71起高版本起好但在网上下得到只是这个版本)以下的方法是对这两种字体美化都起作用的解决方案。

Google一下就能找到其中字体，然后把它们解压后复制到/usr/share/fonts/truetype里面。

1、使用sudo fontconfig-voodoo -f -s zh_CN命令建立连接后，来调整Firefox和一些系统中文字体过度柔化，在修改language-selector.conf文件。

2、sudo gedit /etc/fonts/language-selector.conf

3、找到：

<family>Bitstream Vera Serif</family>下面一行添加：<family>STHeiti</family>

微软雅黑字体就是：<family>Microsoft YaHei</family>

<family>Bitstream Vera Sans</family>下面一行添加：<family>STHeiti</family>

微软雅黑字体就是：<family>Microsoft YaHei</family>

如果两种字体都装那就各使用一个。试过没发现其它问题。

4、调整字体渲染效果

打开“系统－首选项－字体”面板，在“字体渲染”选项中选择“最佳形状”（这个选项由你自己决定，例如LCD可以选择“次象素平滑”）。

5、打开sudo gedit /etc/fonts/fonts.conf文件，找到以<match target="font">的部份（格式最好和它们的一样排列）加入下面一段：

<match target="font">

<test name="family">

<string>STHeiti</string>

<string>Microsoft YaHei</string>

</test>

<edit name="hinting">

<bool>false</bool>

</edit>

</match>

我用的是红色的，如果你是华文黑体，就用黑的那个，两个都用就都用上。

6、也可以使用sudo fc-cache -f -v命令刷新一下字体缓存，，完成。重启x看效果。
